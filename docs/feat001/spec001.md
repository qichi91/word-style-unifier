# Wordスタイル統一Webサービス 要件定義・基本仕様書

## 1. システム概要
本システムは、ユーザーがアップロード（またはリポジトリから取得）したWordファイル（`.docx`）内のテキストスタイルを自動で統一し、ダウンロードできるようにするWebアプリケーションです。特に、表（テーブル）が多用されている文書構造に完全対応します。

### 主な変換ルール
* **半角英数字記号**: フォントAに統一
* **上記以外（日本語等）**: フォントBに統一
* **半角カタカナ**: 全角カタカナに自動変換
* **混在対応**: 1文、あるいは1つのセル内に文字種が混在している場合でも、それぞれのフォントが正しく適用されること。

---

## 2. 技術スタック
* **バックエンドAPI**: FastAPI
* **フロントエンド（画面）**: HTML + Jinja2 テンプレート
* **Word操作ライブラリ**: `python-docx`
* **文字変換ライブラリ**: `jaconv`
* **インフラ/実行環境**: Docker

---

## 3. 機能要件

### 3.1. ファイル入力機能（3系統）
ユーザーは以下の3つの方法のいずれかで変換対象のファイルをシステムに提供します。

1.  **単一ファイルアップロード** (優先度：高)
    * PCローカルから個別の `.docx` ファイルを選択してアップロードする。
2.  **フォルダアップロード** (優先度：高)
    * 複数の `.docx` ファイルが含まれるフォルダを丸ごとアップロードし、一括処理する。
3.  **SVN（Apache Subversion）からの取得** (優先度：**低・後日実装**)
    * SVNリポジトリのリポジトリURLや認証情報を指定し、対象ファイルをサーバー側で直接チェックアウト・取得して処理する。

### 3.2. 変換処理機能（表対応強化）
* **本文（Paragraphs）の処理**: 通常の段落テキストへのフォント適用および半角カナの全角化。
* **表（Tables）の処理**: Wordファイル内のすべてのテーブルを走査し、各セル（Cell）内の段落（Paragraphs）およびラン（Runs）に対して本文と同様のフォント適用・半角カナ全角化を確実に実行する。

### 3.3. ファイル出力機能
* 変換処理が完了した後、画面上に「変換完了通知」と「ダウンロードボタン」を表示する。
* 複数ファイル（フォルダアップロードなど）の場合は、ZIP形式等にアーカイブして一括ダウンロード可能とする。

## 4. 画面（UI）仕様（Jinja2）
シンプルかつ直感的な1〜2画面構成（ダッシュボード形式）とします。

1.  **トップ/入力画面**
    * フォントA（英数字用）、フォントB（日本語用）の名称入力欄（または選択肢）
        * 初期値は代表的なフォントを５種類ほど定義しておく（後でカスタマイズ可能にする）
    * インプット方法の切り替え（「ファイル」「フォルダ」「SVN(グレーアウト等)」）
    * アップロードエリア（ドラッグ＆ドロップ対応）
    * 「変換実行」ボタン
2.  **結果/ダウンロード画面** (同一画面内での非同期遷移、または結果ページ)
    * 処理ステータス（成功 / エラー表示）
    * 「変換済みファイルをダウンロード」ボタン
        * ダウンロードはzip, もしくは7z形式にする

---

## 5. インフラ・デプロイ仕様（Docker）
環境依存を排除し、容易に起動・展開できるようにするため、Dockerコンテナとしてパッケージングします。

* **Dockerfileの構成**:
    * ベースイメージ: `python:3.11-slim` などの軽量Linuxイメージ
    * 必須パッケージ: `python-docx`, `jaconv`, `fastapi`, `uvicorn`, `jinja2`, `python-multipart`
    * （将来用）SVNクライアントツール（`subversion`）のインストール準備
* **docker-compose.yml の用意**:
    * ローカル開発環境での迅速な起動（`docker-compose up --build`）に対応。
    * テンポラリ（一時的）にファイルを保存するアップロード/ダウンロード用ディレクトリのボリュームマウント設定。

---

## 6. 開発ロードマップ（フェーズ分け）

### フェーズ1（最優先）
* Docker環境の構築
* FastAPI + Jinja2 による画面、ファイル/フォルダアップロード機能の実装
* **本文および「表（テーブル）」に対応したWord変換コアロジックの実装**
* ダウンロード機能の実装

### フェーズ2（後続タスク）
* SVN連携機能の追加（認証、リポジトリからのファイル取得ロジックの組込）
* UI/UXのブラッシュアップ（変換進捗バーの表示など）
