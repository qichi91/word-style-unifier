# 機能詳細仕様書: feat003 オンプレvLLM連携による用語揺れ・校正指摘機能

## 1. 目的
本機能は、Wordドキュメント内（本文および表内）のテキストを抽出し、オンプレミス環境に構築された `vLLM` サーバー（LLM）へ送信することで、人間の目では見落としがちな「用語の揺れ」や「誤字脱字」を自動検出し、ユーザーへ指摘・修正案を提示する機能である。

---

## 2. システム構成・接続仕様

### 2.1. インフラストラクチャ
* **LLM推論サーバー**: オンプレミス環境で稼働する `vLLM`
* **インターフェース**: OpenAI互換APIエンドポイント
* **通信プロトコル**: HTTP/HTTPS（FastAPIバックエンドコンテナから社内LAN経由で接続）

### 2.2. 動的稼働制御（ヘルスケック・ガードレール）
外部APIサービスとは異なり、オンプレミス環境のvLLMはメンテナンスやリソースの都合により一時的に停止・切断されている可能性がある。そのため、以下の仕様を必須とする。

1.  **サービス起動時/リクエスト時のヘルスチェック**
    * 変換処理の開始直前に、vLLMの `/v1/models` または `/health` エンドポイントに対して疎通確認（Ping）を行う。
2.  **可用性に応じたアドバンスド・フォールバック**
    * **接続成功時**: Web画面上に「LLM校正機能：有効」と表示し、通常通り処理を実行する。
    * **接続失敗時（タイムアウト含む）**: システム全体のクラッシュを防ぐため、**LLM処理のみを自動的にスキップ（無効化）**する。
    * スキップ時は、ログに警告を出力し、Web画面上には「vLLMサーバーへ接続できないため、スタイル統一処理のみを実行しました」という旨の通知を表示する。

---

## 3. 処理フローとデータ構造

### 3.1. バックエンド処理フロー
1.  Word（`.docx`）から各段落・各セルのテキストを、一意の識別子（`paragraph_id` や `cell_id`）と共に抽出。
2.  vLLMサーバーへの疎通確認。
3.  （接続成功時）抽出したテキストブロックをvLLMへ送信。
4.  vLLMより、JSON形式で解析結果を受信（Structured Outputsのシミュレート）。
5.  解析結果を画面表示用データ、およびWordファイルへのフィードバック用として保持。

### 3.2. vLLMへのプロンプトおよび返却データ構造（JSON）
LLMからのレスポンスを確実に取り扱うため、システムプロンプトで出力フォーマットを厳格に指定（JSON指定）する。

#### システムプロンプト（例）
> あなたはプロの文章校正エディターです。提供されたテキストから「用語の揺れ（例: サーバー/サーバの混在、英数全半角のブレ）」や「明らかな誤字脱字・タイポ」を検出し、必ず以下のJSON配列フォーマットでのみ返却してください。雑談や解説は一切含めないでください。

#### レスポンスJSONスキーマ
```json
[
  {
    "id": "p_001",
    "original": "当システムのサーバ構成について記述する。",
    "detected_error": "サーバ",
    "suggested_fix": "サーバー",
    "reason": "プロジェクト内で『サーバー』表記に統一されている可能性、または揺れの是正。"
  },
  {
    "id": "t_04_c_02",
    "original": "画面項目セツメイの一覧",
    "detected_error": "セツメイ",
    "suggested_fix": "説明",
    "reason": "カタカナ表記の誤字（タイポ）の可能性が高いです。"
  }
]
