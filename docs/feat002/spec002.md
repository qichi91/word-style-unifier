# 機能詳細仕様書: feat002 特定フォーマット対応・テキスト整形

## 1. 目的
本機能は、対象Wordドキュメントが持つ特定の階層構造（セクション構成）を解析し、見出し行の正規化、文字装飾の一括制御（太字・斜体の制御）、およびフォントサイズを統合する機能を提供する。特に、A4横レイアウトや複雑な表構造を維持したまま、テキストのスタイルのみを厳格に統一することを目的とする。

---

## 2. 対象ドキュメントの構造特性
入力されるWordドキュメントは、以下のセクション構成が繰り返し登場するフォーマットとなっている。

| セクションID | セクション名 / コンテンツ | 構造の特徴 | ページレイアウト |
| :--- | :--- | :--- | :--- |
| **X** | 画面名（題名） | 本文（見出し段落） | A4 縦 |
| **X1** | セクション1（英訳・ウィンドウタイトル等） | 2列5行の表（Table） | A4 縦 |
| **X2** | 概要 | 1列2行の表（Table） | A4 縦 |
| **X3** | 画面レイアウト | 1列2行の表（Table） ※2行目に画像添付 | A4 縦 |
| **X4** | 画面項目説明 | **10列N行の複雑な表（Table）** | **A4 横** |
| **X5** | 注釈 | 2列N行の表（Table） | A4 縦 |

* ※ `X` は任意の章番号（例: 1, 2, 3...）とし、通常 `X8` 程度まで連続する。
* ※ サブ画面の存在に応じて、`X3` および `X4` のセットは同一章内で複数回記述されるケースがある。

---

## 3. スタイル統一・整形ルール

### 3.1. フォントサイズルール
* **適用範囲**: 本文の段落、およびすべての表（Table）の中のテキスト。
* **仕様**: 一律で **`10.5pt`** に統一する。

### 3.2. 文字装飾（太字・斜体）ルール
* **太字（Bold）の適用**:
    * **セクション見出し行のみ**太字とする（判定基準は「3.4」を参照）。
    * それ以外の本文、および表（Table）の中のテキストは**すべて太字を解除**する。
* **斜体（Italic）の適用**:
    * ドキュメント内のすべてのテキスト（見出し、本文、表内含む）について、**斜体を一律解除（なし）**とする。

### 3.3. スペース正規化ルール（見出し行のみ）
* セクション見出しの識別子（「X」「X1.」「X2.」など）の直後にある空白文字を、**半角スペース1個**に強制統一する。
* セクション見出しの識別子と次の行には改行を入れないこと
* セクション間には改行を入れる。
* 連続する余分な空白（全角スペースや複数の半角スペース）はすべて除去・置換する。
    * *修正前例*: `X3.   画面レイアウト` 
    * *修正後例*: `X3. 画面レイアウト`

### 3.4. セクション見出しの判定アルゴリズム
以下の正規表現パターンにマッチする段落（Paragraph）を「セクション見出し」として認識する。
* **パターン**: `^(X\d*下位階層表記?\.?)\s+(.*)$`
* **注意点**: 表（Table）のセル内に存在するテキストは、このパターンにマッチした場合であっても**見出しとはみなさず、太字解除の対象**とする（誤判定防止ルーチン）。

---

## 4. テクニカルガードレール（実装上の制約・注意点）

### 4.1. ページレイアウト（セクション方向）の維持
`X4 画面項目説明` は10列の表を格納するために「A4横レイアウト」が採用されている。Pythonコード側でスタイルや段落プロパティを上書きする際、Wordの `Section` オブジェクト（ページ設定データ）を破壊・初期化してはならない。**ページ方向（縦・横）には一切干渉せず、Run単位の文字プロパティのみを走査・変更すること。**

### 4.2. 添付画像の保護
`X3 画面レイアウト` の表内（2行目）には画面キャプチャ等の画像がインライン（`InlineShape`）で埋め込まれている。テキストの走査および置換、サイズ変更処理を行う際、これらの画像要素を削除・破損させずに維持すること。

### 4.3. 複数回登場する表への対応
`doc.tables` を用いた全走査ロジックを実装し、X3, X4がドキュメント内に複数回ランダムに登場する場合でも、インデックス依存ではなく登場順にすべてを網羅して処理できる構造にすること。
